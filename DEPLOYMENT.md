# V19+ Production Deployment Guide

Deploy stack:

| Layer | Platform | URL example |
|-------|----------|-------------|
| Database | **Supabase** (PostgreSQL) | — |
| API | **Render** | `https://v19plus-api.onrender.com` |
| Consumer app | **Vercel** | `https://v19plus-web.vercel.app` |
| Admin CMS | **Vercel** (2nd project) | `https://v19plus-admin.vercel.app` |
| Mobile APK | **Capacitor** (Android) | wraps the Vercel web app |

---

## Architecture

```
┌─────────────┐     /api proxy      ┌──────────────┐
│ Vercel Web  │ ──────────────────► │ Render API   │
│  (Next.js)  │                     │  (NestJS)    │
└─────────────┘                     └──────┬───────┘
┌─────────────┐     /api proxy             │
│ Vercel Admin│ ───────────────────────────┤
└─────────────┘                            ▼
┌─────────────┐                     ┌──────────────┐
│ Android APK │ ── loads web URL ──►│  Supabase    │
│ (Capacitor) │                     │  PostgreSQL  │
└─────────────┘                     └──────────────┘
```

Vercel rewrites `/api/*` → Render so auth cookies stay same-origin.

## Step 1 — Supabase (Database)

1. Go to [supabase.com](https://supabase.com) → **New project**
2. Choose region close to your users (e.g. Mumbai/Singapore for India)
3. Save the database password
4. **Project Settings → Database → Connection string**
5. Copy two strings:

| Variable | Supabase mode | Port |
|----------|---------------|------|
| `DATABASE_URL` | **Transaction pooler** | 6543 |
| `DIRECT_URL` | **Session / Direct** | 5432 |

Append `?pgbouncer=true&connection_limit=1` to the pooled URL.

6. Locally, create `.env` from `.env.example` and paste both URLs
7. Push schema and seed:

```bash
cd V19plus-main
npm install
npm run db:push --workspace=apps/backend
npm run db:seed --workspace=apps/backend
```

Optional (recommended for production): create a migration:

```bash
cd apps/backend
npx prisma migrate dev --name init
git add prisma/migrations
git commit -m "Add initial Prisma migration"
```

## Step 2 — Render (Backend API)

1. Push code to **GitHub**
2. [render.com](https://render.com) → **New → Blueprint** → connect repo
3. Render reads `render.yaml` and creates `v19plus-api`
4. Set these **Environment Variables** (Dashboard → Service → Environment):

```env
DATABASE_URL=<supabase pooled url>
DIRECT_URL=<supabase direct url>
FRONTEND_URL=https://YOUR-WEB.vercel.app
ADMIN_URL=https://YOUR-ADMIN.vercel.app
GOOGLE_CLIENT_ID=<from Google Cloud>
GOOGLE_CLIENT_SECRET=<from Google Cloud>
GOOGLE_REDIRECT_URI=https://YOUR-API.onrender.com/api/auth/google/callback
STRIPE_SECRET_KEY=sk_live_xxx
STRIPE_WEBHOOK_SECRET=whsec_xxx
RAZORPAY_KEY_ID=rzp_live_xxx
RAZORPAY_KEY_SECRET=<secret>
SMTP_HOST=smtp.gmail.com
SMTP_USER=<email>
SMTP_PASS=<app password>
COOKIE_SAME_SITE=lax
```

`JWT_ACCESS_SECRET` and `JWT_REFRESH_SECRET` are auto-generated by Render.

5. Deploy and verify:

```
https://YOUR-API.onrender.com/api/health
→ {"status":"ok","service":"v19plus-api",...}
```

6. **Stripe webhook**: Dashboard → Webhooks → Add endpoint:
   ```
   https://YOUR-API.onrender.com/api/subscription/webhook
   ```

> **Note:** Render free tier sleeps after inactivity. First request may take ~30s.

## Step 3 — Vercel (Consumer Web App)

1. [vercel.com](https://vercel.com) → **Add New Project** → import GitHub repo
2. Settings:
   - **Root Directory:** leave as repo root (`.`)
   - **Framework:** Next.js (auto-detected via `vercel.json`)
3. **Environment Variables:**

```env
BACKEND_URL=https://YOUR-API.onrender.com
NEXT_PUBLIC_API_URL=/api
NEXT_PUBLIC_GOOGLE_CLIENT_ID=<same as GOOGLE_CLIENT_ID>
NEXT_PUBLIC_SOCKET_URL=https://YOUR-API.onrender.com
```

4. Deploy → note URL: `https://YOUR-WEB.vercel.app`
5. Update Render env `FRONTEND_URL` to this URL and redeploy API

---

## Step 4 — Vercel (Admin Panel)

Create a **second Vercel project** from the same repo.

1. **Root Directory:** `.` (repo root)
2. Override build settings (or use `apps/admin/vercel.json`):
   - Build: `npm run build --workspace=apps/admin`
   - Output: `apps/admin/.next`
3. **Environment Variables:**

```env
BACKEND_URL=https://YOUR-API.onrender.com
NEXT_PUBLIC_API_URL=/api
```

4. Deploy → note URL: `https://YOUR-ADMIN.vercel.app`
5. Update Render `ADMIN_URL` to this URL and redeploy API

---

## Step 5 — Google OAuth (required for login)

[Google Cloud Console](https://console.cloud.google.com/apis/credentials) → OAuth 2.0 Client:

**Authorized JavaScript origins:**
```
https://YOUR-WEB.vercel.app
http://localhost:3000
```

**Authorized redirect URIs:**
```
https://YOUR-API.onrender.com/api/auth/google/callback
http://localhost:4000/api/auth/google/callback
```

## Step 6 — Android APK (Capacitor)

The APK wraps your **live Vercel URL** — no need to bundle Next.js statically.

### Prerequisites

- [Android Studio](https://developer.android.com/studio) installed
- JDK 17+
- Android SDK

### Setup

```bash
cd V19plus-main
npm install

# Set your deployed web URL
set CAPACITOR_SERVER_URL=https://YOUR-WEB.vercel.app   # Windows CMD
# $env:CAPACITOR_SERVER_URL="https://YOUR-WEB.vercel.app"  # PowerShell

cd apps/web
npm run mobile:init      # first time only — creates android/ folder
npm run mobile:sync      # sync Capacitor config
npm run cap:android      # opens Android Studio
```

### Build APK in Android Studio

1. **Build → Generate Signed Bundle / APK**
2. Choose **APK**
3. Create or select a keystore (save it securely)
4. Build **release** variant
5. APK output: `apps/web/android/app/build/outputs/apk/release/`

### Install on phone

Transfer the APK to your Android device and enable "Install from unknown sources", or use:

```bash
adb install app-release.apk
```

### Update `capacitor.config.ts`

Replace the placeholder URL:

```ts
server: {
  url: 'https://YOUR-WEB.vercel.app',
  cleartext: false,
}
```

Or set `CAPACITOR_SERVER_URL` env before `cap sync`.

## Environment variable cheat sheet

| Variable | Where | Value |
|----------|-------|-------|
| `DATABASE_URL` | Render | Supabase pooler (6543) |
| `DIRECT_URL` | Render | Supabase direct (5432) |
| `FRONTEND_URL` | Render | Vercel web URL |
| `ADMIN_URL` | Render | Vercel admin URL |
| `BACKEND_URL` | Vercel (both) | Render API URL (no `/api`) |
| `NEXT_PUBLIC_API_URL` | Vercel (both) | `/api` |
| `GOOGLE_REDIRECT_URI` | Render | `https://api.onrender.com/api/auth/google/callback` |
| `CAPACITOR_SERVER_URL` | Local build | Vercel web URL |

## Deployment order (summary)

```
1. Supabase     → create DB, copy connection strings
2. Local        → db:push + db:seed against Supabase
3. Render       → deploy API, set env vars
4. Vercel Web   → deploy consumer app
5. Vercel Admin → deploy admin panel
6. Render       → update FRONTEND_URL + ADMIN_URL, redeploy
7. Google OAuth → add production origins + redirect URI
8. Stripe       → add webhook URL
9. Capacitor    → build signed APK
```

## Troubleshooting

| Problem | Fix |
|---------|-----|
| API health 503 on Render | Check `DATABASE_URL` / `DIRECT_URL`; view Render logs |
| Login works locally, not prod | Verify Google OAuth origins; check `FRONTEND_URL` on Render |
| API calls fail on Vercel | Ensure `BACKEND_URL` is set; check Render service is awake |
| Admin 401 errors | Set `ADMIN_URL` on Render; redeploy API |
| APK blank screen | Set `CAPACITOR_SERVER_URL` to live Vercel URL before `cap sync` |
| Cookies not persisting | Keep `NEXT_PUBLIC_API_URL=/api` (proxy mode); `COOKIE_SAME_SITE=lax` |
